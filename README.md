# 3x-ui Reverse Proxy Setup

Автоматическая установка и настройка nginx как reverse proxy для [3x-ui](https://github.com/MHSanaei/3x-ui) с поддержкой VLESS транспортов и SSL.

## Поддерживаемые протоколы

| Протокол | Транспорт | Через Nginx |
|----------|-----------|-------------|
| VLESS + WebSocket | WS | ✅ |
| VLESS + gRPC | gRPC | ✅ |
| VLESS + XHTTP (SplitHTTP, mode: packet-up) | XHTTP | ✅ |
| VLESS + Reality (uTLS: qq, dest: apple.com:443) | TCP/XTLS-Vision | ❌ прямое подключение |

## Поддерживаемые ОС

- Ubuntu 22.04 / 24.04
- Debian 12 / 13

## Что делает скрипт

- Включает **TCP BBR**
- Отключает **UFW**
- Устанавливает **nginx** (reverse proxy)
- Получает SSL сертификат через **acme.sh** (Let's Encrypt)
- Устанавливает **3x-ui** (опционально, если не установлен)
- Настраивает порт, URI path и credentials панели
- Автоматически создаёт все 4 inbound'а в 3x-ui через API
- Устанавливает **fail2ban** (без кастомных jails — только базовая защита)
- Генерирует случайные пути (5–12 символов) для всех inbound'ов

## Использование

```bash
bash <(curl -Ls https://raw.githubusercontent.com/drno88/reverse-proxy-3x-ui/main/setup.sh)
```

> Запускать от root.

## Что спросит скрипт

1. **Домен** — должен иметь A-запись на IP сервера (нужен для SSL)
2. **Проксировать панель через Nginx?** — если нет, доступ только через SSH tunnel
3. **Путь к панели** — если 3x-ui уже стоит, подтягивается из БД автоматически
4. **Путь и порт** для каждого транспорта (WS / gRPC / XHTTP)
5. **Порт** для Reality (прямое подключение, без nginx)

> Если 3x-ui уже установлен — порт и путь панели читаются из БД, credentials не меняются.

## Архитектура

```
Клиент → 443 (HTTPS) → Nginx → localhost:порт → Xray inbound
                                                  ├── WS       :10001
                                                  ├── gRPC     :10002
                                                  └── XHTTP    :10003

Клиент → Reality порт (прямое) → Xray inbound (Reality)
```

> Reality требует прямого TLS — nginx в цепочке не участвует.

## После установки

Все inbound'ы создаются **автоматически** через API 3x-ui. Панель сразу готова к работе.

При свежей установке логин и пароль генерируются и показываются в конце скрипта.

## fail2ban

Устанавливается с настройками по умолчанию — без кастомных jails.  
При необходимости кастомные jails можно раскомментировать в `install_fail2ban()`.
